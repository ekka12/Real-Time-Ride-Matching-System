# Real-Time-Ride-Matching-System
Real-Time Ride Matching System built using ASP.NET Core (version 6.0) and SQL Server(22).

This project demonstrates a scalable backend architecture for a ride booking platform where riders can request rides and nearby available drivers are matched in real time using geolocation-based distance calculation.

Key Features:
- Rider and Driver Registration/Login
- JWT Authentication & Authorization
- Driver Live Location Updates
- Real-Time Ride Request Processing
- Nearby Driver Matching using Latitude/Longitude
- Ride Assignment Logic
- Mail Inbox Notification System
- SQL Server Stored Procedure Driven Backend
- Swagger API Testing Support

Tech Stack:
- ASP.NET Core Web API
- SQL Server
- Entity Framework Core
- JWT Authentication
- Swagger/OpenAPI
- Stored Procedures
- Geolocation Distance Formula (Great Circle/Haversine)

The solution focuses on clean backend design, asynchronous processing concepts, scalability considerations, and production-minded architecture.
